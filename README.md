# Pre-commit Настройка

После клонирования репозитория выполните следующие шаги для настройки pre-commit в директории auth_service:

## 1. Установка pre-commit
Перейдите в директорию auth_service и установите pre-commit хуки:

```bash
cd auth_service
poetry run pre-commit install
```

## 2. Первоначальная проверка (опционально)
Чтобы сразу проверить и исправить все существующие файлы, выполните:

```bash
poetry run pre-commit run --all-files
```

Это просканирует все файлы и автоматически исправит проблемы с форматированием или ошибки линтера.

## 3. Рабочий процесс
После установки pre-commit будет автоматически запускаться перед каждым коммитом:

- Если будут найдены и автоматически исправлены проблемы, коммит будет заблокирован
- Добавьте исправленные файлы обратно в staging: `git add .`
- Закоммитьте снова: `git commit -m "Ваше сообщение"`
